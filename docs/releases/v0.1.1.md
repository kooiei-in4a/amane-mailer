## Included fixes since v0.1.0

- Unknown and duplicate JSON properties are rejected before request acceptance.
- Provider exceptions are classified and sanitized before persistence, logging,
  and Admin UI display.
- Release publish workflows validate the release tag and refuse to overwrite
  existing GHCR tags.
- Local compose fresh data directories are initialized for Linux/macOS Docker hosts.
- Release smoke and public evidence docs were updated after the initial release.
- Admin audit logging and log-forging sanitization fixes from current `main` are included.

Production ACS users should prefer this release over `v0.1.0` because it includes
the provider-error sanitization path.

## Known limitations

- Published runtime image platform is `linux/amd64` only.
- SQLite deployment assumes a single node / single replica.
- Admin UI remains disabled by default, internal-network-only, and experimental.
- `202 Accepted` means the request was persisted; provider delivery completes asynchronously.
