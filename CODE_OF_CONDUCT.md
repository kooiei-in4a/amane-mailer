# Code of Conduct

Amane Mailer aims to be a practical, respectful, and security-conscious open
source project.

## Expected Behavior

- Be kind, patient, and specific.
- Assume good intent while still being clear about risks and mistakes.
- Keep discussions focused on the project and its users.
- Report security-sensitive information privately instead of posting it in
  public issues, pull requests, or discussions.

## Unacceptable Behavior

- Harassment, threats, or discriminatory language.
- Sharing private information without permission.
- Publishing secrets, credentials, exploit details, or private operational data
  in public project spaces.
- Repeatedly derailing issue or pull request discussions.

## Enforcement

Project maintainers may edit, hide, or delete comments; close issues or pull
requests; restrict participation; or take other moderation action when needed
to keep the project healthy.

Security concerns should use the process described in `SECURITY.md` and GitHub
private vulnerability reporting.
